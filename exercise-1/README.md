## Learn Basics of g++ CLI

- Compile the TCP client and server using `g++` from command line.
  <br> Done, both server and client working.
- What are the most important command line arguments to learn for `g++`?
  <br> -c, -o, -S, -Wall, -Wextra, -g, -O0, -O2, -std=c++x, -fsanitize
- What is the difference between debug vs release versions?
  <br> Debug versions contain debugging information and symbols in the binary which help during debugging (eg: using gdb) and also don't use compiler optimizations. Release versions are stripped of all such information and use compiler optimization to get the fastest and smallest builds.
- What are the tradeoffs between debug and release versions?
  <br>Debug - Larger build size, faster compile time due to no optimizations, easy to use for debugging
  <br>Release - Smaller build size, slow compilatation, cannot be used for debugging
- What arguments would you use in a debug build?
  <br>-g -O0 -Wall -Wextra -fsanitize=address -fsanitize=undefined
- What about for release?
  <br>-O2 -Wall -Wextra
  <br>(-O3 sometimes increases binary size so -O2 is moslty preferred)
- What other kinds of build types are useful?
  <br> Debug and release seem to be the two main kinds of build modes. But we can consider assembly output, object file output as different modes. Assembly output can be used to check the low level code generated by the compiler and get the understanding of what is happening at the lowest level. Object file mode can be used to create object files of all the files in a project and then link them later with a single command.
  <br><br> On looking up further after answering the questions, I found out that we can consider more build modes such as RelWithDebInfo which is optimized but still has debug symbols and used for profiling.
